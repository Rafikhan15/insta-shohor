Question: How does javascript work?

Answer: JavaScript is a client-side scripting language and one of the most popular scripting languages all over the world. The term .client-side scripting language means that it runs at the client-side inside the web-browsers but one important thing to remember is that client's web-browser also needs to support the JavaScript or it must be JavaScript enabled. Nowadays it's getting more and more popular teams are leveraging script support on many levels in their stack-front-end, back-end, hybrid apps embedded devices and much more.

Question: What does javascript event loop do?

Answer: Javascript is single-threaded it executes only one operation at a time. The fact that only one operation is executed at a time on a single thread is what makes javascript synchronous. As a result it's could clearly slow down our applications. To avoid this situation javascript has a concurrency model based on the event loop that provides it with the ability to process multiple tasks asynchronously. Js event loop which is responsible for executing the code, collecting and processing events, and executing queued sub-tasks.
